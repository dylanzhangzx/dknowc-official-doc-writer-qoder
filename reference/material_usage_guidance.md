# 召回材料使用指导

本文件用于指导 Agent 将深知搜索召回材料转化为公文正文。它不是固定模板，不要求每篇文章按同一结构写作；目标是保证材料使用准确、稳妥、服务用户任务。

## 基本原则

1. 用户信息优先。用户提供的任务背景、单位名称、时间地点、对象范围、工作要求和本地口径优先级最高。搜索材料只能补充和支撑，不得覆盖用户明确给出的事实。
2. 观点先行，材料支撑。正文应围绕用户任务组织观点和结构，再选择材料支撑观点；不得把搜索结果简单拼接成正文。
3. 分类使用，不混用来源。政策、数据、案例和表述各有用途，不得把外地案例写成本地现状，不得把上级政策写成本单位已出台政策。
4. 宁可稳妥，不要硬编。没有明确出处、时间或数据口径时，不写具体数字；可改用“持续推进”“逐步完善”“加快形成”等稳妥表述，或提示需要用户确认。
5. 表达吸收，不照搬。可学习政策材料和案例材料的公文表达、结构和措辞，但不得大段复制原文，也不得让正文显得像资料摘编。
6. 正文纯净，溯源分离。正式公文正文只呈现成稿内容；普通 Word 保留固定 AI 生成提示，红头 Word 不保留 AI 生成提示；素材使用情况、知识专库链接和需核验信息单独写入 HTML 素材来源说明。

## 四类材料如何使用

### 政策依据型

用于正文中的背景依据、工作方向、必要性、合法性和上级要求。

使用方式：
- 国家级政策可用于说明总体方向、战略要求和上位依据。
- 本省/本市政策可用于说明本地部署、工作边界和具体要求。
- 政策标题不宜密集堆砌；通常选择最关键的 1-3 个政策作为支撑即可。
- 正文中可概括政策精神，不必每次都写完整标题。

禁止：
- 将外省政策写成本省政策依据。
- 将征求意见稿、解读稿、新闻稿写成已经正式实施的规范性文件。
- 将政策中的鼓励性方向改写成用户单位必须已经完成的事实。

### 数据支撑型

用于现状判断、问题分析、成效描述、趋势研判和目标合理性。

使用方式：
- 只有材料明确给出数字、时间、地域和口径时，才写具体数据。
- 多个数据冲突时，优先使用国家级、本地权威部门、最新正式发布数据。
- 数据只服务于判断，不要在正文中堆砌过多统计口径。
- 对缺少权威数据但方向明确的内容，可使用趋势性表述。

禁止：
- 根据不完整材料自行推算、补齐或扩展数据。
- 将全国数据、本省数据、本市数据混写为同一层级结论。
- 将旧年份数据写成当前年度最新情况。
- 将来源不明、口径冲突或时间不匹配的数据写成确定结论。

### 参考案例型

用于借鉴做法、机制设计、工作举措和实施路径。

使用方式：
- 外地案例应抽象为可借鉴做法，如机制建设、平台支撑、场景开放、部门协同、试点推进。
- 案例材料更适合进入“工作建议”“下一步安排”“可借鉴经验”等部分。
- 写入正文时优先转化为通用做法，不必保留过多地名和项目细节。

禁止：
- 把外地案例写成本地已开展工作。
- 把企业案例、试点案例写成普遍成效。
- 为了显得丰富而堆砌多个地区案例。

### 表述参考型

表述参考只来自已召回材料，不单独搜索。

使用方式：
- 学习正式材料中的句式、层次、术语和稳妥表达。
- 用于优化语气和公文风格，使正文更像正式机关材料。
- 可参考“总体看、围绕、聚焦、持续推进、不断完善、加快形成”等表达。

禁止：
- 将表述参考当作事实来源。
- 在素材使用情况中把表述参考单独列为搜索材料。
- 为追求公文味而堆砌空话、套话，导致内容变虚。

## 写作前的内部整理

执行过搜索后，正式写正文前应在内部完成以下判断，不需要向用户完整展示：

- 用户给定的硬约束有哪些，哪些不得被搜索材料覆盖。
- 哪些政策可作为依据，哪些只能作为背景或参考。
- 哪些数据可以写具体数值，哪些只能写趋势判断。
- 哪些案例可以转化为做法建议，哪些不适合进入正文。
- 是否存在地域、时间、政策效力或数据口径风险。

如果上述判断中出现关键缺口或冲突，应按搜索异常和素材不足规则暂停，请用户确认是否补搜、调整口径或基于现有材料继续。

## 正文中的使用方式

- 通知、函、提醒函等短文种：优先满足事项表达，材料只用于必要依据，不宜展开论证。
- 报告、总结、方案、汇报材料等长篇材料：材料应服务于章节观点，可按“背景依据、现状成效、问题短板、工作建议”灵活嵌入。
- 政策研究、产业研究、调研分析：可更多使用政策和数据支撑判断，但仍要区分事实、趋势和建议。
- 如用户明确要求“不引用外省政策”，外地材料只能作为内部启发，不进入正文依据。

## 事实口径分级

写入正文前，按风险分级处理事实材料：

- 可直接使用：正式政策原文、本级权威部门数据、用户提供并明确要求采用的事实。
- 谨慎使用：公开报道、部门动态、解读材料、案例材料，可作为背景或趋势，不宜写成正式依据。
- 降级处理：文号、发布日期、精确数值、排名、金额、占比、全国首个/唯一/领先等信息无法交叉确认时，正文改用概括表达。
- 不进入正文：来源缺失、口径冲突、时间越界、地域不匹配或明显超出用户任务边界的信息。

被降级处理但仍有参考价值的信息，应写入素材来源说明的“需人工核验信息”，提醒用户正式报送前核验。

## 长篇材料的结构化使用

对工作总结、工作要点、实施方案、专项整治方案、会议讲话、研讨发言、汇报材料等长篇材料，写作前应先内部确定结构，不要直接把材料摘要顺序拼成正文。

常用结构：

- 总结/报告类：总体情况、主要成效、问题不足、下一步工作。
- 工作要点类：总体要求、重点任务、组织保障。
- 方案/专项整治类：工作目标、工作重点、阶段安排、责任分工、工作要求。
- 讲话/发言类：会议目的、形势判断、问题分析、重点部署、责任落实。

每个章节应先明确段落功能，再选择材料支撑：

- 背景依据段：使用政策依据和用户任务背景。
- 成绩成效段：只使用用户提供或搜索材料明确支撑的事实和数据。
- 问题短板段：以用户提供的问题为主，搜索材料只能辅助归纳，不得替用户单位定性。
- 工作部署段：将政策要求、案例做法和用户目标转化为任务、措施、责任和时限。
- 保障落实段：围绕组织领导、协同机制、督导考核、闭环销号、长效机制展开。

表达参考只能用于优化语气和节奏，不得替代事实支撑。若正文某段只有排比句、口号或抽象要求，没有具体任务和执行抓手，应重写该段。

## 生成后自检

正文生成后，应快速检查：

- 是否有搜索材料覆盖用户口径的问题。
- 是否误用外地政策或案例。
- 是否写入未证实数据、过期数据或自行推算数据。
- 是否把材料堆成资料摘编，而不是围绕用户任务成文。
- 是否存在政策标题过多、案例过多、套话过多的问题。
- 正式公文正文是否未内嵌素材使用情况、知识专库链接或长 URL。
- 素材来源说明 HTML 是否只列实际使用的写作素材，不单列“表述参考型”。
- 长篇材料是否已经形成“目标-任务-措施-责任/时限-保障”的闭环。
- 小标题是否能概括段落内容，未出现标题正式但正文空泛的问题。
