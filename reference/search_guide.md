# 搜索流程详细指南

本文件包含搜索流程中的格式模板和输出示例，供 AI 执行搜索相关步骤时参考。

---

## 一、搜索结果分类输出格式

所有搜索完成后，AI 对全部素材按以下格式分类输出：

```
【搜索结果分析】

素材统计：共召回 X 篇文章，Y 个相关段落

分类结果：

**1. 政策依据型**
- 《XX管理办法》（文号）- 可引用条款
- 《XX实施细则》（文号）- 可引用条款
  → 共 N 篇

**2. 参考案例型**
- XX省做法：...
- XX市经验：...
  → 共 N 篇

**3. 数据支撑型**
- 统计数据：XX年达到XX
- 调研结果：...
  → 共 N 个数据点

说明：表述参考型不作为独立搜索类型，不单独列入搜索方案。需要学习表达时，只能从已召回的政策依据、数据支撑、参考案例材料中吸收行文方式。

素材充足度评估：
- 政策依据：充足 / 较少 / 缺失
- 参考案例：充足 / 较少 / 缺失
- 数据支撑：充足 / 较少 / 缺失
- 表述参考：不单独搜索，从已召回材料中吸收
```

## 二、搜索方案展示模板

执行深知搜索前，先向用户展示搜索方案并等待确认。用户可见的搜索方案只写检索方向、地域、素材类型、检索内容和用途，不展示脚本命令、参数或文件名。

禁止在用户可见方案中出现以下内容：

- `--area`、`--search-type`、`--policy`、`--search-channel`、`--clean`、`--output` 等脚本参数
- “参数”列或“参数：...”字段
- `python3 scripts/dkag_search.py ...` 形式的命令
- 输出文件名、JSON 文件名或内部保存路径

```
【搜索方案】

搜索地域：任务所在地区 / 中国

1. 政策依据
- 地域：某省
- 检索内容：某省人工智能产业政策支持
- 用途：查找可作为本省政策依据的文件、意见、办法等

2. 数据支撑
- 地域：某省
- 检索内容：某省人工智能产业发展规模、产业数据和工作进展
- 用途：查找政府公开发布的产业数据、工作进展和成效

3. 参考案例
- 地域：中国
- 检索内容：人工智能产业发展典型做法和应用案例
- 用途：查找可借鉴的政府部门案例和经验做法，不作为本省政策依据

说明：表述参考不单独搜索，将从上述材料中吸收公文表达和结构。
```

用户确认后再执行搜索；如用户调整地域、query 或素材类型，按调整后的方案执行。执行时必须把每条搜索项的“搜索目的”作为内部 `--purpose` 传给搜索脚本，用于后续知识专库链接显示名；不得把 `--purpose` 展示给用户。

内部执行时，Agent 再自行选择对应脚本参数并调用 `scripts/dkag_search.py`。内部参数选择不得要求用户理解，也不得作为搜索方案的一部分展示。

## 三、参考范文召回流程

### 步骤1：向用户推荐

```
【参考范文推荐】

在搜索素材中发现一篇文章对成文结构有重要参考价值：

《文章标题》
来源：XX，作者，日期
参考价值：[说明参考什么，如"文章结构与我们的报告高度吻合，可学习其论述逻辑"]

确认获取全文作为参考范文吗？（是/否）
```

### 步骤2：用户确认后，用文章标题全文召回

```bash
python3 scripts/dkag_search.py "[文章完整标题]" --full
```

注意：不加 `--clean`！加了会丢弃"全文"字段。

### 步骤3：从返回结果中提取目标文章

API 会返回多篇全文，必须按标题精确匹配，只提取目标文章的"全文"字段：

```python
for article in result['content']['data']['检索文章']:
    if article['文章标题'] == '目标文章标题':
        full_text = article['全文']
        break
```

### 步骤4：确认提取成功后告知用户

```
【参考范文已获取】

《文章标题》
来源：XX，作者，日期
字数：约XXXX字
已提取全文，将在撰写时作为参考范文使用。
```

提取失败时告知用户，回到步骤1重新选择或跳过。

## 四、素材交付指引模板

进入撰写环节前，AI 将所有素材整理成以下结构传入写作模型：

```
## 素材使用指引

### 一、写作素材

以下素材用于填充正文内容，请根据标注的用途在对应部分使用。

#### 1. 政策依据
用途：[说明用于正文的哪个部分]
- 《XX办法》（文号，数据源：XX，日期）
- 《XX意见》（文号，数据源：XX，日期）

#### 2. 数据支撑
用途：[说明用于哪个部分]
- 2023年市场规模5059.5亿元（数据源：XX）
- 注册无人机387万架（数据源：XX）

#### 3. 参考案例
用途：[说明用于哪个部分]
- XX省做法：XXX（数据源：XX）

#### 4. 表述参考
用途：从上述已召回材料中学习公文用语风格，直接融入正文，不需要引用；不得作为独立检索结果列出

### 二、参考范文（如有）

以下文章作为成文结构参考，学习其章节安排和论述逻辑，但不要照搬内容。

《文章标题》
- 来源：XX，作者，日期
- 字数：约XXXX字
- 参考价值：[具体说明参考什么]
- 注意事项：[如时效性、适用范围等]
- 全文：[粘贴提取的全文内容]
```

## 五、素材来源说明 HTML

执行过搜索时，正式公文 Word 不在正文末尾添加素材清单和知识专库链接。撰写完成后，单独生成 `标题_素材来源说明.html`，用于展示可溯源依据。素材来源说明固定为 HTML，不生成 Word 版。

生成方式：

1. 先将素材来源说明整理为结构化 JSON，保存到 `official-docs/input/标题_素材来源说明.json`。
2. 再调用 `scripts/source_note_html.py` 生成 HTML：

```bash
python3 scripts/source_note_html.py official-docs/input/标题_素材来源说明.json --output 标题_素材来源说明.html
```

JSON 示例：

```json
{
  "title": "深圳市人工智能赋能政务服务应用情况调研报告_素材来源说明",
  "summary": "本说明用于回看本次深知搜索召回材料，核验正文中的政策、数据和案例依据。正式正文不嵌入链接，需核验时可通过知识专库链接回看原始材料。",
  "knowledge_bases": [
    {
      "label": "深圳AI与智慧城市政策依据",
      "url": "https://yun.dknowc.cn/wlcb/ShenZhi-policy/#/knowledgebase/?id=XXX"
    },
    {
      "label": "深圳政务智能审批改革数据",
      "url": "https://yun.dknowc.cn/wlcb/ShenZhi-policy/#/knowledgebase/?id=XXX"
    }
  ],
  "materials": [
    {
      "type": "政策依据型",
      "material_name": "《深圳经济特区人工智能产业促进条例》",
      "source": "深圳市人大常委会",
      "date": "2024年",
      "section": "一（二）深圳地方政策体系",
      "support": "支撑深圳以地方立法推动人工智能产业和政务场景应用的制度基础。",
      "verify": "正式报送前核对法规名称、施行日期和条款表述。"
    },
    {
      "type": "数据支撑型",
      "material_name": "“深小i”智能问答服务数据",
      "source": "深圳市政务服务和数据管理局公开报道",
      "date": "2026年",
      "section": "三（一）“深小i”AI政务助手",
      "support": "支撑“深小i”服务量、应答率、一次解决率等成效描述。",
      "verify": "相关数据时效性较强，正式使用前建议核对最新口径。"
    }
  ],
  "checks": [
    "政策文件文号、发布日期建议与正式发文文件核对。",
    "精确数据建议以主管部门最新发布口径为准。"
  ]
}
```

格式要求：
- 正式公文正文中不得包含 `【素材使用情况】`、`【知识专库链接】`、知识专库 URL 或素材清单。
- 素材来源说明 HTML 是辅助溯源文件，不是正式公文附件，不附在正式正文后。
- HTML 首页必须先展示知识专库链接，再展示素材使用卡片和需人工核验信息。
- 每条知识专库链接必须使用对应搜索项的“搜索目的”作为 `label`，优先读取搜索结果 JSON 的 `search_meta.purpose`；如缺失，再使用该次搜索的 query 概括。
- `materials` 中每条素材至少包含 `material_name`、`source`、`section`、`support`；存在数据口径、发布日期、文号或强判断时，应补充 `verify`。
- 每条素材只写实际用于正文的核心依据，不机械罗列所有搜索结果；同类材料可合并说明，避免来源说明变成流水账。
- 不得使用表格承载素材使用情况；HTML 由脚本生成卡片式布局。
- 不得使用 `→ 用于：`、箭头符号或其他内部工作流标记。
- 写作素材和参考范文分开列出。
- 参考范文只写 `参考用途：` 和 `使用边界：`，不得把表述参考混入写作依据。
- 知识专库链接按地域或素材类型分类列出
- 知识专库链接必须逐条复制自搜索结果 JSON 顶层 `knowledgeBase` 字段；如顶层缺失，再读取原始返回位置 `content.knowledgeBase`。该链接不在 `content.data` 的文章材料字段内，不得因为 `data` 中找不到就认定缺失。
- HTML 由 `scripts/source_note_html.py` 生成；不要手写完整 HTML 页面，也不要用 Word 排版脚本生成素材来源说明。
