# 搜索流程详细指南

本文件包含搜索流程中的格式模板和输出示例，供 AI 执行搜索相关步骤时参考。

---

## 一、搜索结果分类输出格式

所有搜索完成后，AI 对全部素材按以下格式分类输出：

```
【搜索结果分析】

素材统计：共召回 X 篇文章，Y 个相关段落

分类结果：

**1. 政策依据型**
- 《XX管理办法》（文号）- 可引用条款
- 《XX实施细则》（文号）- 可引用条款
  → 共 N 篇

**2. 参考案例型**
- XX省做法：...
- XX市经验：...
  → 共 N 篇

**3. 数据支撑型**
- 统计数据：XX年达到XX
- 调研结果：...
  → 共 N 个数据点

说明：表述参考型不作为独立搜索类型，不单独列入搜索方案。需要学习表达时，只能从已召回的政策依据、数据支撑、参考案例材料中吸收行文方式。

素材充足度评估：
- 政策依据：充足 / 较少 / 缺失
- 参考案例：充足 / 较少 / 缺失
- 数据支撑：充足 / 较少 / 缺失
- 表述参考：不单独搜索，从已召回材料中吸收
```

## 二、搜索方案展示模板

执行深知搜索前，先向用户展示搜索方案并等待确认。每条搜索项需同时写清楚 query、地域、素材类型和拟使用参数：

```
【搜索方案】

搜索地域：任务所在地区 / 中国

1. 政策依据
- query：某省人工智能产业政策支持
- 地域：某省
- 参数：--search-type policy --policy
- 用途：查找可作为本省政策依据的文件、意见、办法等

2. 数据支撑
- query：某省人工智能产业发展规模 2025
- 地域：某省
- 参数：--search-type govSite --search-channel govSearch
- 用途：查找政府公开发布的产业数据、工作进展和成效

3. 参考案例
- query：人工智能产业发展 典型做法
- 地域：中国
- 参数：--search-type govSite --search-channel govSearch
- 用途：查找可借鉴的政府部门案例和经验做法，不作为本省政策依据

说明：表述参考不单独搜索，将从上述材料中吸收公文表达和结构。
```

用户确认后再执行搜索；如用户调整地域、query 或素材类型，按调整后的方案执行。

## 三、参考范文召回流程

### 步骤1：向用户推荐

```
【参考范文推荐】

在搜索素材中发现一篇文章对成文结构有重要参考价值：

《文章标题》
来源：XX，作者，日期
参考价值：[说明参考什么，如"文章结构与我们的报告高度吻合，可学习其论述逻辑"]

确认获取全文作为参考范文吗？（是/否）
```

### 步骤2：用户确认后，用文章标题全文召回

```bash
python3 scripts/dkag_search.py "[文章完整标题]" --full
```

注意：不加 `--clean`！加了会丢弃"全文"字段。

### 步骤3：从返回结果中提取目标文章

API 会返回多篇全文，必须按标题精确匹配，只提取目标文章的"全文"字段：

```python
for article in result['content']['data']['检索文章']:
    if article['文章标题'] == '目标文章标题':
        full_text = article['全文']
        break
```

### 步骤4：确认提取成功后告知用户

```
【参考范文已获取】

《文章标题》
来源：XX，作者，日期
字数：约XXXX字
已提取全文，将在撰写时作为参考范文使用。
```

提取失败时告知用户，回到步骤1重新选择或跳过。

## 四、素材交付指引模板

进入撰写环节前，AI 将所有素材整理成以下结构传入写作模型：

```
## 素材使用指引

### 一、写作素材

以下素材用于填充正文内容，请根据标注的用途在对应部分使用。

#### 1. 政策依据
用途：[说明用于正文的哪个部分]
- 《XX办法》（文号，数据源：XX，日期）
- 《XX意见》（文号，数据源：XX，日期）

#### 2. 数据支撑
用途：[说明用于哪个部分]
- 2023年市场规模5059.5亿元（数据源：XX）
- 注册无人机387万架（数据源：XX）

#### 3. 参考案例
用途：[说明用于哪个部分]
- XX省做法：XXX（数据源：XX）

#### 4. 表述参考
用途：从上述已召回材料中学习公文用语风格，直接融入正文，不需要引用；不得作为独立检索结果列出

### 二、参考范文（如有）

以下文章作为成文结构参考，学习其章节安排和论述逻辑，但不要照搬内容。

《文章标题》
- 来源：XX，作者，日期
- 字数：约XXXX字
- 参考价值：[具体说明参考什么]
- 注意事项：[如时效性、适用范围等]
- 全文：[粘贴提取的全文内容]
```

## 五、素材来源说明格式

执行过搜索时，正式公文 Word 不在正文末尾添加素材清单和知识专库链接。撰写完成后，单独生成 `标题_素材来源说明.docx`，用于展示可溯源依据。只有用户明确要求纯文本或 Markdown 时，才可改为 `.md`。建议使用以下内容：

```markdown
# 标题_素材来源说明

【素材使用情况】

一、写作素材

1. 《民用航空法》（数据源：全国人大，2026年修订）
   用途：现状概述"政策体系"部分，支撑"国家层面法律保障"论述

2. 2023年市场规模5059.5亿元（数据源：中国电子信息产业发展研究院）
   用途：现状概述"产业规模"部分

3. 某省集聚全国30%以上产业链企业（数据源：某省政府网站）
   用途：现状概述"地方实践"部分，作为典型案例

二、参考范文

1. 《我国低空经济发展面临的问题与政策建议》（来源：国家信息中心，张晓兰，2024年12月30日）
   参考：文章整体三段式结构（现状、问题、建议），以及"问题-建议"一一对应的论述逻辑

【知识专库链接】

国家级政策：
https://yun.dknowc.cn/wlcb/ShenZhi-policy/#/knowledgebase/?id=XXX

某省：
https://yun.dknowc.cn/wlcb/ShenZhi-policy/#/knowledgebase/?id=XXX

【需人工核验信息】

1. 某项政策文号或发布日期：正文已采用概括表述，正式报送前建议核验原文。
2. 某项统计数据：不同来源口径不完全一致，正式使用前建议以主管部门最新发布为准。
```

格式要求：
- 正式公文正文中不得包含 `【素材使用情况】`、`【知识专库链接】`、知识专库 URL 或素材清单。
- 素材来源说明不是正式公文附件，除非用户明确要求“附在正文后”。
- 每条素材标注"用途：正文具体部分"，不得使用 `→ 用于：`、箭头符号或其他内部工作流标记。
- 写作素材和参考范文分开列出
- 素材使用情况只保留：文章标题/数据内容、来源、用途，不需要内容摘要
- 标题必须原样使用 `【素材使用情况】`、`【知识专库链接】` 和 `【需人工核验信息】`，不得写成普通标题或省略方括号
- 知识专库链接按地域或素材类型分类列出
- 知识专库链接必须逐条复制自原始搜索结果 JSON 的 `knowledgeBase` 字段，不得手写、猜测、改写或混用其他搜索链接
- Markdown 草稿中可保留原始 URL；生成 Word 时，排版脚本会将 URL 转为“分类标题（点击打开）”的可点击文本，避免长链接直接铺满页面。
